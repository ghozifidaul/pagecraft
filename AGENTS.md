# OpenCode Instructions for PageCraft

**Do not create commits unless the user explicitly asks.**

## Packages

| Directory | What | Entrypoint / Scripts |
|-----------|------|---------------------|
| `pagecraft-api/` | Hono Worker (Cloudflare) | `src/index.ts` — mounts 3 routers under `/api/art-styles`, `/api/books`, `/api/books` (pages router shares prefix). Routes → `src/routes/`, DB → `src/db/`, logic → `src/services/`, helpers → `src/lib/`. |
| `pagecraft-ui/` | Vite + React 19 + Tailwind v4 + react-router v8 | `src/main.tsx`. Dev proxies `/api` → `http://localhost:8787`. Tailwind v4 uses `@tailwindcss/vite` plugin (no PostCSS config). |

## Commands

Run all commands inside the respective package dir.

### API (`pagecraft-api/`)
- `npm run dev` — `wrangler dev`
- `npm run deploy` — `wrangler deploy --minify`
- `npm run cf-typegen` — sync Worker bindings (run after editing `wrangler.jsonc`)
- `npm run test` — unit tests only
- `npm run test:integration` — requires `GEMINI_API_KEY` in `.env` (silently skipped otherwise via `it.skipIf`)
- `npm run test:all` — all test suites

### UI (`pagecraft-ui/`)
- `npm run dev` — Vite dev server
- `npm run build` — `tsc -b && vite build`
- `npm run lint` — ESLint

## Architecture & Gotchas

- **Hono bindings**: Always type with `<{ Bindings: CloudflareBindings }>`. Requires `worker-configuration.d.ts` (regenerated by `cf-typegen`).
- **Gemini**: Story uses `gemini-3.1-flash-lite` (structured JSON output), illustration uses `gemini-3.1-flash-image` (image output). Both via Interactions API (`v1beta/interactions`).
- **D1**: Binding `pagecraft_db`. Schema in `pagecraft-api/migrations/0001_create_books_and_pages.sql`. Cascade `books → pages`, image cleanup in R2 is caller's responsibility.
- **R2**: Bucket `pagecraft-images`, binding `IMAGE_BUCKET`. Path: `books/{bookId}/pages/{pageId}/{timestamp}.png`. Signed URLs (1h) via S3-compatible API — requires `R2_ACCESS_KEY_ID`, `R2_SECRET_ACCESS_KEY`, `R2_ENDPOINT` env vars.
- **CORS**: Reads `FRONTEND_URL` env var as allowed origin.
- **No auth** — intentionally anonymous.
- **Art style images** live in `pagecraft-api/public/art-styles/` (static Worker assets).
- **No CI workflows** exist.
- **TypeScript v6** throughout.
- Test setup (`vitest.config.ts`) loads `.env` via `dotenv`. Timeout: 30s. Tests in `__tests__` dirs next to source.
- Unit tests mock `globalThis.fetch` (Gemini stub). Route tests mock all DB and service modules with `vi.mock()`.

## Routes

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/art-styles` | List art presets |
| GET | `/api/books` | List all books |
| POST | `/api/books` | Create book + generate story |
| GET | `/api/books/:id` | Get book with pages |
| DELETE | `/api/books/:id` | Delete book (cascades pages + R2 images) |
| PUT | `/api/books/:id/pages/:pageId/story` | Manual story edit |
| POST | `/api/books/:id/pages/:pageId/story/regenerate` | AI story regen with feedback |
| POST | `/api/books/:id/pages/:pageId/illustration` | Generate illustration (sequential order enforced) |
| POST | `/api/books/:id/pages/:pageId/illustration/regenerate` | Regen illustration with feedback |
